# QueryVault Usage Guide

QueryVault is a Retrieval-Augmented Generation (RAG) tool that allows you to interactively chat with your documents using LLaMA models (via Groq) and Google's Generative AI Embeddings directly from your terminal.

## 1. Prerequisites
- Python 3.8+
- Create a virtual environment (recommended):
  `python -m venv .venv`
- Activate the virtual environment:
  - Windows: `.venv\Scripts\activate`
  - Mac/Linux: `source .venv/bin/activate`

## 2. Install Dependencies
Install the required packages using pip:
`pip install python-dotenv langchain-groq langchain-google-genai langchain-chroma langchain-community pypdf`

## 3. Setup APIs
You need to set up your API keys to use the embedding and generation models.
1. Create a `.env` file in the root directory (where this `how_to_use.txt` is located).
2. Look at `.samplegit` for reference on what API keys are needed.
3. Add your actual API keys from Groq and Google AI Studio into the `.env` file.

## 4. Load & Index Documents
Before you can ask questions, you must load and process your document into the Vector Database (Chroma).
1. Open `backend/load_document.py` and update the `file_path` to point to your target PDF document.
2. Run the vectorization script to process the document and save embeddings:
   `python backend/data_vectorDb.py`
This will create a local folder `chroma_langchain_db` containing your vector data.

## 5. Ask Questions
To interactively chat with your processed document, run the LLM invoker script:
`python backend/llm_invoke.py`

This will start an interactive prompt in your terminal. Type your question and hit enter to get a context-aware answer based on your document. Type `exit` to stop.
